List：使用频率最高的集合。可以用于存储有序数据，比如查询数据库表等操作。
Set：可以与数学上的集合相提并论，即元素不可重复，强调元素独一无二。
Queue：队列，先进先出。可实现需要”排队“的业务逻辑。
Map：“深不可测”的图。实际上大部分情况理解为键值对就够用。 ^rHfsYxtp

为什么没有Stack栈呢？
早期Stack类通过继承Vector类，但后期推荐使用Queue接口下的
Deque接口，也就是双向队列 + 个人限制来实现Stack。 ^NUBaK4rS

尽量避免使用Vector类。因为其加锁特性导致的性能问题以及设计过时，使得
在性能或安全方面都有更加成熟的解决方案。
性能：ArrayList
安全：Collections.synchronizedList(new ArrayList<>()) ^vIB8qHv3

Iterator 迭代器的使用：
1.其实Iterator才是最通用的实践，原因在于它作为List的上层接口，
实现了检查逻辑。比起增强for，普通for多了删除元素这一功能。
2.Iterator更像指针，我们可以通过hasNext()检查，next()提取。
Iterator<String> it = list.iterator();
while (it.hasNext()) {
    if (it.next().equals("B")) {
        it.remove();
    }
}
3.增强for循环是调用Iterator的，并且因为代码更简洁易读，在
无需删除元素的情况下，使用会更友好。
4.若没有获取索引需求（绝大部分情况下都无需获取）建议不要使用普通for循环。 ^zILuEfwT
