Spring Cache的使用：
前言：关于缓存这里使用Redis，实际上更基础的用法是使用RedisTemplate去与底层数据结构进行接触。
但Spring对缓存进行封装后只剩下了这几个基本操作： ^lqpLAu7j

1.@EnableCaching  加在启动类 ^z9rIXeuV

2.@Cacheable(cacheName = " ",key = " ")  要获取数据先访问缓存，没有则访问数据库；数据库将数据返回到应用之后，再将数据异步写入缓存。 ^tAQ0HevD

3.@CachePut(cacheName = " ",key = " ") 将数据库返回值放到缓存 ^8fKER9DQ

4.@CacheEvict(cacheName = " ",key = " ") 或是 @CacheEvict(cacheName = " ",allEntries = true)
删除缓存cacheName中关于key的数据或清空cacheName中的所有数据 ^Q6GIPURK

key可以有多种写法，一般是取方法中某对象/参数的属性或变量。一般来说就是key = "#id"或是"#object.property"（抽象形式）
cacheName是对于缓存业务场景的命名，便于分类。比如菜品缓存和订单缓存，这样相当于分了个文件夹，清空菜品缓存时对订单缓存没有影响。 ^bwFCPheP

---初识缓存，只要是工具，那么就是做给人用的。RedisTemplate的封装已经十分简便，但Spring又给了最基础的注解操作。因此我们在
实际工作中只用更关注业务逻辑，并处理好缓存更深层次的问题，如缓存雪崩，缓存击穿，缓存穿透等经典场景，或是分布式锁等最佳实践。 ^1bOaUyHi

十分注意：SpringCache默认情况下是没有设置缓存过期时间的，所以我们可以在配置文件/AOP进行设置。 ^IF72yNYP
